## Repository 1

**Repository Name: opentopodata**  
**Link: https://github.com/ajnisbet/opentopodata**  

### Purpose
This project provides an overview, installation instructions, and
usage documentation for an elevation data API called Open Topo Data.

### Repository Structure
There is a requirements text file. Additionally, there is a "datasets"
folder within the "docs" folder istead of within the "data" folder.

### Documentation
I think the README is very helpful and provides a clear overview of
how to get started and use Open Topo Data.

### Activity
It was most recently updated last month.

### Key Observation
There are code snipet examples with each step in the README which 
makes it very easy to follow the instructions.

## Repository 3

**Repository Name: awesome-gis**  
**Link: https://github.com/sshuair/awesome-gis**  

### Purpose
This project provides a collection of gis related resources, tools, news,
communities, and more.

### Repository Structure
This repository does not have a "docs" folder and instead has the two .md
files in the main repository.

### Documentation
I think the README is very helpful. It is well oragnized and broken up by
the category of the resources. The README is almost too long, however, and
it might be better to break each category into different files or folders 
with the README just being a general overview of what awesome-gis is for.

### Activity
Yes it was last updated 3 weeks ago and is open for anyone to contribute.

### Key Observation
There is a document that outlines contributing guidelines which seems like
a standard of practice.

## Reflection

In this exercise, I learned that the structure of open source projects
varies widely. For the most part, all of the README files provided very
detailed information and there were some consistencies between folder
names like "docs", "scripts", and so on. Depending on whether the project
takes contributions from the community determines how much more documentation
is required. Each of the projects also seems to have a way to track issues,
feedback, or other questions/comments that users may have.
